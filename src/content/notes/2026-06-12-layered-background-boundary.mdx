## 现象

首页左侧带有轻微档案标尺和色调层，宽屏时颜色在固定位置突然结束。暗色模式对比更高，因此边界比浅色模式更明显。

## 根因

问题不在渐变色标本身，而在背景图层使用了固定宽度并关闭横向重复。浏览器准确地绘制了一个有限尺寸的背景层，超出该尺寸后回到页面底色。

```css
background-size: 42rem 100%;
background-repeat: no-repeat;
```

当视口宽度大于 `42rem`，图层右边缘就成为清晰的垂直分界线。

## 修复原则

修复没有继续扩大一个“足够大”的固定值，而是让负责整体氛围的图层覆盖完整视口，只让坐标、标尺等局部装饰保留有限范围。

```css
background-size:
  100% 100%,
  24px 24px;
background-repeat:
  no-repeat,
  repeat;
```

## 为什么暗色更容易暴露

两个相邻表面的可见差异可以简化为：

<p class="formula" role="math">boundary visibility ∝ | layer luminance - page luminance |</p>

暗色背景中，只要两个深色值的亮度与色相不同，连续的大面积区域就会放大边缘。浅色模式不是没有问题，只是差值更小。

## 回归检查

- 桌面宽屏检查背景是否覆盖完整。
- 390px 检查装饰层是否造成横向滚动。
- 单独切换暗色主题，不用浅色结果代替。
- 检查 `scrollWidth` 与 `clientWidth` 是否一致。
- 控制台保持 0 error / 0 warning。
