## Project brief

ScholarLens AI V2 是一个面向个人日常使用的本地 PDF 阅读工作区。它不是对 V1 的直接重写，而是重新围绕一条可恢复、可验证的主流程组织系统：

```text
导入 PDF
-> 检查页数与文件约束
-> 选择本地解析或远程 OCR
-> 规范化页面与内容块
-> 对照阅读 PDF / Markdown
-> 翻译、笔记与文档上下文 AI
```

第一目标是可靠的本地个人阅读体验，而不是多用户平台、云同步或复杂插件系统。

## Reader workspace

当前阅读界面已经形成稳定的三栏结构：

| 区域 | 职责 |
| --- | --- |
| 左栏 | 原始 PDF 页面，保持为主要事实来源 |
| 中栏 | 规范化 Markdown / Translation 切换 |
| 右栏 | Chat / Note 独立开关与当前上下文 |

PDF 页、Markdown 页面和内容块之间保留轻量同步。单击用于选择 Chat / Note 上下文，双击用于跨栏定位，避免阅读操作和导航操作互相干扰。

## Parsing and OCR

V2 当前提供两条解析路径：

- `pymupdf_native`：本地提取文本、图片、基础表格和阅读顺序，不上传文档。
- `zhipu_glm_ocr`：通过 GLM-OCR 处理复杂版面、公式、图表和扫描型内容。

两条路径最终写入同一份 provider-independent normalized contract，使阅读器、翻译、笔记和 AI 不直接依赖原始 provider 输出。

GLM-OCR 执行已经从单次整文档调用扩展为：

- 按页数与平均文件体积选择 5 / 10 / 20 页物理分块
- 最多 6 个独立任务并发
- 已完成分块渐进发布，允许提前开始阅读
- 保留成功分块，仅重试失败或缺失范围
- 合并时恢复原始页码、图片资产与空白页槽位

## Translation, notes and AI

- Translation 使用版本化 prompt、内容块 ID、缓存身份和渐进结果，保留原始页面与内容块引用。
- Notes 按文档保存，可关联页面、内容块、OCR provider 和结果 hash，并支持 Markdown 导出。
- Reading Assistant 默认使用无网络 mock 路径，也可连接 OpenAI-compatible provider。
- Chat 上下文支持文档概览、当前页和选中内容块，不要求用户每次先做精确选择。
- Settings 只保存非敏感配置；真实 API key 不进入公开内容或浏览器日志。
