# 00_项目开发计划书

版本：v1.0  
项目：Pixel Archive  
位置：`docs/00_项目开发计划书.md`  
用途：定义 Pixel Archive 的项目定位、建设目标、页面范围、技术路线、MVP 边界、阶段计划和后续路线图。  

---

## 1. 项目概述

**Pixel Archive** 是一个个人网站项目。

它不是传统简历页，也不是普通博客模板，而是一个带像素风气质的个人档案馆，用于长期存放和展示：

```txt
项目作品
技术笔记
开发日志
阶段性存档
视觉 / 像素 / Minecraft / 图像实验内容
```

项目核心定位：

> Pixel Archive 是一个以极简阅读体验为底层、以像素组件为识别符号、以复古终端 / 存档隐喻为氛围的个人档案馆。

---

## 2. 项目目标

Pixel Archive 的目标不是一次性做成复杂网站，而是先搭建一个稳定、可扩展、可长期维护的个人网站框架。

第一阶段目标：

```txt
先让网站跑起来
先确定页面结构
先建立视觉系统基础
先建立内容系统
先建立开发规范
先放规范占位内容
后续再逐步填充真实内容
```

最终目标：

```txt
Home 有独特入口感
Projects 能展示项目与作品
Notes 能沉淀文章与学习笔记
Archive 能记录重要阶段
Lab 能放实验性内容
About 能提供极简个人入口与联系方式
整体风格统一，长期可维护
```

---

## 3. 项目不是做什么

Pixel Archive 初版不做以下事情：

```txt
不做传统个人简历站
不做求职导向页面
不做纯博客模板
不做后台管理系统
不做数据库
不做登录系统
不做评论系统
不做复杂动态应用
不做完整游戏化网页
不做全站复古操作系统仿站
```

项目第一阶段应保持轻量，不主动引入超出当前目标的功能。

---

## 4. 核心设计理念

项目视觉和内容组织遵循：

```txt
极简做骨架
像素做识别
复古做氛围
内容可长期沉淀
结构可持续扩展
```

理解方式：

```txt
极简负责：阅读、留白、信息层级、长期维护
像素负责：按钮、边框、标签、首页识别、404、Lab
复古负责：archive、save slot、terminal、save point 等隐喻
```

网站应当有个人风格，但不能牺牲阅读体验和信息清晰度。

---

## 5. 目标内容类型

Pixel Archive 初版规划四类核心内容：

```txt
Projects
Notes
Archive
Lab
```

### 5.1 Projects

用于展示项目和作品。

包括但不限于：

```txt
ScholarLens AI
GLM-OCR 工具
舵机通用测试平台
个人网站本身
飞控 / AI / 工具类项目
```

项目可以是：

```txt
WIP
Done
Prototype
Paused
```

允许半成品出现，但必须清楚标注状态。

---

### 5.2 Notes

用于沉淀文章、学习笔记、开发日志。

内容类型包括：

```txt
Article：正式文章
Log：阶段记录 / 开发日志
Note：学习笔记 / 临时整理 / 知识卡片
```

Notes 页面必须保证长文阅读体验，不能过度像素化。

---

### 5.3 Archive

用于记录重要阶段和项目存档点。

例如：

```txt
Started Pixel Archive
ScholarLens AI 阶段完成
毕业设计阶段收尾
某个重要工具完成原型
```

Archive 不是日记，不记录太碎的日常内容。

---

### 5.4 Lab

用于放实验性内容。

包括：

```txt
像素实验
UI 组件实验
Minecraft 建筑
图像生成
小型交互 Demo
视觉风格测试
```

Lab 是最自由的页面，允许比其他页面更有视觉实验性。

---

## 6. 页面范围

初版导航固定为：

```txt
Home / Projects / Notes / Archive / Lab / About
```

同时实现：

```txt
404 页面
```

---

## 7. 页面职责

## 7.1 Home

首页是 Pixel Archive 的入口，不是个人介绍页。

首页应包含：

```txt
Pixel Archive 标题
Archive Terminal
游戏菜单式入口
3 个 Save Slots
Status Bar
简短 UI 文案
轻微像素点缀
```

首页不应包含：

```txt
大段个人简介
真实姓名作为主视觉
简历式内容
求职导向文案
```

首页气质：

> 像打开一个私人像素档案终端。

---

## 7.2 Projects

Projects 页面用于展示项目。

初版要求：

```txt
项目列表
项目分类
标签展示
Featured Projects
项目卡片
项目详情页
状态 Badge
```

项目少时也应能自然展示，不应显得空。

---

## 7.3 Notes

Notes 页面用于展示文章和笔记。

初版要求：

```txt
Notes 列表
Article / Log / Note 类型
分类
标签
文章详情页
Markdown / MDX 正文
图片
代码块
公式支持
目录 TOC
```

搜索功能后续再做。

---

## 7.4 Archive

Archive 页面用于展示存档点和阶段记录。

初版要求：

```txt
按年份分组
按月份分组
像游戏存档点
静态列表
```

节点不需要详情页，后续可扩展。

---

## 7.5 Lab

Lab 页面用于展示实验内容。

初版要求：

```txt
空壳页面
精选卡片结构
允许更强像素风
允许局部 Neo-Brutalist 感
```

真实内容后续逐步填充。

---

## 7.6 About

About 页面保持克制。

初版只需要：

```txt
极短介绍
联系方式占位
GitHub 链接占位
```

不要做正式简历页。

---

## 7.7 404

404 页面要有 Pixel Archive 气质。

概念：

```txt
lost archive
save file not found
```

必须提供返回首页入口。

---

## 8. 技术路线

初版技术栈：

```txt
Astro
TypeScript
MDX
Astro Content Collections
原生 CSS
CSS Variables
```

技术原则：

```txt
静态优先
内容优先
依赖克制
无后端
无 CMS
无数据库
本地先跑通
后续部署到 Cloudflare Pages
```

不使用：

```txt
Tailwind CSS
shadcn/ui
完整第三方 UI 框架
React / Vue UI 组件库
后端服务
数据库
CMS
复杂动画库
```

外部 UI / CSS / Astro 模板仅作参考，不作为直接依赖。

---

## 9. 内容管理路线

内容通过 Markdown / MDX 文件维护。

推荐内容目录：

```txt
src/content/
├─ projects/
├─ notes/
├─ archive/
└─ lab/
```

图片统一放在：

```txt
public/images/
├─ projects/
├─ notes/
├─ archive/
├─ lab/
└─ shared/
```

内容维护方式：

```txt
本地 Markdown / MDX
Git 管理版本
后续可从 Obsidian 迁移
不做在线 CMS
```

## 10. 部署路线

初版路线：

```txt

Phase 2：页面结构和视觉系统成型
Phase 3：放入少量占位内容
Phase 4：推到 GitHub
Phase 5：Cloudflare Pages 部署
```

## 12. 第一阶段完成标准

第一阶段完成后应满足：

```txt
1. npm run dev 可以启动
2. npm run build 可以通过
3. 首页显示 Pixel Archive
4. 首页不是个人简介页
5. 导航包含 Home / Projects / Notes / Archive / Lab / About
6. 主题切换按钮存在
7. 浅色 / 暗色主题基础可用
8. Projects / Notes / Archive / Lab / About / 404 页面存在
9. 404 使用 lost archive / save file not found 概念
10. src/content/config.ts 定义四个 collections
11. 占位内容命名符合规范
12. README 已更新
13. WORKLOG 已更新
14. 没有引入 Tailwind / UI 框架 / CMS / 数据库
15. 页面风格符合 Pixel Archive 基本方向
```

## 16. 最终原则

> Pixel Archive 的开发应先建立稳定、清晰、可扩展的个人网站框架，再逐步填充真实内容和高级功能。  
> 第一阶段的核心不是“大而全”，而是“结构干净、风格明确、内容系统稳定、构建可通过”。
