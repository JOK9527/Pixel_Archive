# 12_视觉参考资源清单

版本：v1.0  
项目：Pixel Archive  
位置：`docs/references/12_视觉参考资源清单.md`  
用途：整理 Pixel Archive 可参考的外部视觉资源，包括像素风、终端风、Neo-Brutalist、极简正文、Astro 模板和资源合集。本文档仅作为参考资料，不是强制开发规范。  

---

## 1. 本文档的作用

本文档负责回答：

```txt
1. Pixel Archive 可以参考哪些外部视觉资源？
2. 每个资源适合参考什么？
3. 哪些资源是主参考，哪些只是局部参考？
4. 这些资源不能怎样使用？
5. Agent 在视觉开发时如何正确使用参考资源？
```

本文档不负责定义最终视觉规则。  
最终视觉规则以：

```txt
docs/standards/03_视觉风格规范.md
```

为准。

---

## 2. 使用原则

所有参考资源只用于：

```txt
观察视觉语言
参考组件形态
参考页面结构
参考排版方法
参考暗色模式
参考交互细节
```

不允许：

```txt
直接整站套模板
直接引入完整 UI 框架
大量复制第三方 CSS
直接照搬配色
直接照搬组件库视觉
让网站看起来像某个现成模板
因此改变技术栈
因此引入 Tailwind / shadcn/ui / React UI 库
```

Pixel Archive 的最终实现应使用：

```txt
Astro
TypeScript
原生 CSS
CSS Variables
自研组件
```

---

## 3. 参考资源分级

资源分为三档：

```txt
A 档：主参考
B 档：局部参考
C 档：收藏 / 检索入口
```

含义：

| 等级 | 含义 |
|---|---|
| A 档 | 对 Pixel Archive 第一阶段视觉方向有直接指导意义 |
| B 档 | 可参考局部组件、边框、排版或氛围 |
| C 档 | 作为以后查找资源、比较方案和扩展功能时使用 |

## 4. A 档主参考

## A1. NES.css

类型：

```txt
Pixel / 8-bit CSS framework
```

参考重点：

```txt
像素按钮
像素边框
对话框
进度条
容器
复古游戏 UI 的轻量表达
```

适合用于：

```txt
Button 的 pixel 变体
Card 的 pixel 变体
Save Slot 边框
404 页面
Archive 存档点
Lab 卡片
```

不能用于：

```txt
整站直接引入 NES.css
全站都做成 NES 游戏 UI
Notes 正文像素化
照搬全部按钮和对话框样式
```

Pixel Archive 使用方式：

```txt
只参考像素边框和短 UI 组件气质。
最终组件应使用项目内自研 CSS 实现。
```

## 10. 色彩参考原则

Pixel Archive 不直接采用任何参考库的原始配色。

最终配色应遵守：

```txt
米白 / 暖灰白背景
深灰正文
低饱和紫色主识别
低饱和绿色强调
浅灰 / 淡紫灰边框
柔和深灰暗色模式
```

禁止：

```txt
NES 原色系大面积使用
Windows 98 蓝灰系统感全站使用
黑底荧光绿终端全站使用
Neo-Brutalist 高饱和撞色全站使用
```

---

## 11. 字体参考原则

参考库中的字体不一定适合直接使用。

Pixel Archive 字体原则：

```txt
正文使用系统默认字体
代码使用 monospace
像素 / monospace 只用于 Logo、Badge、Save Slot、StatusBar 等点缀
正文不使用像素字体
```

不要因为参考 NES.css 就把全站字体像素化。

## 13. Agent 使用参考资源规则

Agent 做视觉任务时应遵守：

```txt
1. 先读 03_视觉风格规范.md
2. 再读本文件
3. 明确当前页面应该参考哪些资源
4. 只借鉴局部特征
5. 不直接引入框架
6. 不改变技术栈
7. 不复制完整主题
8. 不让参考资源覆盖 Pixel Archive 主风格
```

如果 Agent 想引入某参考库作为依赖，必须先说明理由，并等待用户确认。
