# 03_视觉风格规范

版本：v1.1
项目：Pixel Archive  
位置：`docs/standards/03_视觉风格规范.md`  
用途：定义 Pixel Archive 的整体视觉方向、风格平衡原则、色彩系统、字体系统、页面风格权重、组件视觉边界、动效规则、暗色模式规则和视觉验收标准。  

---

## 1. 本文档的作用

本文档负责回答：

```txt
1. Pixel Archive 应该长什么样？
2. 极简、像素、复古终端之间怎么平衡？
3. 哪些外部风格可以参考，哪些不能直接套用？
4. 色彩、字体、边框、阴影、圆角如何控制？
5. 每个页面的像素浓度应该是多少？
6. 哪些组件可以像素化，哪些必须克制？
7. 动效和暗色模式应该遵守什么原则？
8. 视觉完成后如何验收？
```

本文档不负责定义：

```txt
技术栈与依赖管理
内容文件命名
模块依赖方向
组件 props 细节
文案写法
部署流程
测试命令
```

这些由其他规范负责。

---

## 2. 视觉一句话定义

Pixel Archive 的视觉方向是：

> 一个以极简阅读体验为底层、以像素组件为识别符号、以复古终端 / 存档隐喻为氛围的个人档案馆。

它不是：

```txt
普通博客模板
传统个人简历站
纯像素游戏网站
Windows 98 仿站
黑底绿字终端站
高饱和 Neo-Brutalist 网站
```

它应该是：

```txt
干净
克制
有秩序
有像素识别
有存档终端气质
适合长期阅读
适合展示项目
适合沉淀内容
```

---

## 3. 风格总原则

Pixel Archive 的风格不是把极简、像素、复古平均混合。

正确关系是：

```txt
极简做骨架
像素做识别
复古做氛围
```

建议比例：

```txt
极简：70% 负责结构、留白、阅读、信息层级
像素：20% 负责按钮、边框、图标、首页识别
复古：10% 负责终端感、存档感、轻微旧系统氛围
```

目标效果：

```txt
有像素风，但不幼稚
有复古感，但不模板化
有极简感，但不无聊
有个人风格，但不影响信息阅读
```

---

## 4. 风格平衡规则

## 4.1 极简负责骨架

极简主要负责：

```txt
页面留白
文字层级
阅读宽度
信息结构
项目展示
响应式布局
长文阅读
明暗主题稳定性
```

以下区域必须以极简和可读性为主：

```txt
Notes 正文
项目详情正文
About 正文
公式
代码块
图表说明
长表格
长段落
```

这些区域不应过度使用像素字体、强边框、重阴影或终端效果。

---

## 4.2 像素负责识别

像素风主要用于界面识别，而不是铺满正文。

适合明显像素化的区域：

```txt
首页 Archive Terminal
Save Slots
主菜单入口
按钮
Badge
Tag
Status Pill
项目卡片边框
Archive 存档点
404 页面
Lab 页面
Footer 短句
```

像素风的目标是让用户感到“这是 Pixel Archive”，而不是让所有页面变成游戏 UI。

---

## 4.3 复古负责氛围

复古元素主要通过隐喻实现。

可以使用：

```txt
archive online
save loaded
new slot unlocked
save point created
lost archive
save file not found
> Projects
> Notes
SAVE SLOT 01
terminal
cursor
status bar
```

不要使用：

```txt
全站 Windows 98
全站 NES 游戏菜单
全站黑底绿字终端
全站扫描线
强 CRT 扭曲
高饱和复古游戏配色
复古系统窗口全套照搬
```

## 6. 页面风格权重

不同页面的像素浓度不同，不允许全站平均用力。

| 页面 | 极简 | 像素 | 复古终端 | 风格说明 |
|---|---:|---:|---:|---|
| Home | 40% | 35% | 25% | 风格最明显，像打开 Pixel Archive |
| Projects | 65% | 25% | 10% | 信息展示优先，卡片可像素化 |
| Notes | 85% | 10% | 5% | 阅读体验优先，正文必须克制 |
| Archive | 55% | 25% | 20% | 可以明显体现存档点概念 |
| Lab | 45% | 35% | 20% | 最自由，可使用更多实验性边框和卡片 |
| About | 80% | 15% | 5% | 克制、短、干净 |
| 404 | 40% | 35% | 25% | 可以更有趣，使用 lost archive 概念 |

## 7. 色彩系统

## 7.1 色彩方向

Phase 10 验收后的稳定配色方向：

```txt
梨花白 / 月白 / 白雪藤构成明亮纸面背景
黛蓝 / 墨蓝承担标题和正文结构
亚丁绿承担品牌、在线和建设状态
朱红承担 active、箭头、动作链接和小像素焦点
琥珀承担存档、时间、锁定和 index ready
藕荷紫仅承担 Lab、Mode Quiet 和少量次级状态
中性蓝灰结构线承担普通边框
墨青黑 / 蓝灰表面构成独立暗色主题
```

视觉效果应偏：

```txt
柔和
克制
安静
有质感
彩色点缀小而清楚
不过度赛博
不过度游戏化
```

色彩面积保持：

```txt
背景纸色：65%–75%
表面色：15%–25%
文字与结构线：7%–10%
彩色点缀：2%–4%
```

---

## 7.2 色彩角色

| 色彩角色 | 用途 |
|---|---|
| Background | 梨花白主背景，允许月白 / 白雪藤微冷变化 |
| Surface | Hero、卡片、终端和面板的纸白表面 |
| Surface Raised | 重点卡片和状态栏的高一级表面 |
| Heading | 黛蓝大标题，暗色模式使用暖纸白 |
| Text Primary | 墨蓝正文 |
| Text Muted | 日期、说明、弱信息 |
| Border | 低对比中性蓝灰结构线 |
| Border Strong | 主容器、时间轴和局部重点结构 |
| Primary | 亚丁绿；品牌、online、Done、building |
| Accent | 朱红；active、focus、箭头、LOAD 动作 |
| Warm | 琥珀；save point、locked、日期和 archive |
| Violet | Lab、Mode Quiet 和少量次级角标 |
| Danger | 错误或危险状态 |
| Code Background | 代码块背景 |
| Selection | 文本选中背景 |


## 8. 字体系统

## 8.1 字体原则

```txt
正文使用系统默认字体，稳定优先
代码使用 monospace
Logo / Badge / Save Slot / 小状态可以使用像素字体或 monospace
长文标题不要过度像素化
正文绝不使用像素字体
```

---

## 8.2 字体使用边界

可以使用像素 / monospace 点缀：

```txt
Logo
Badge
Tag
Save Slot
按钮短文本
Status Bar
Archive Terminal
404 页面短句
小标题装饰
```

不要使用像素字体：

```txt
Notes 正文
项目详情正文
About 正文
长段落
公式解释
代码说明
图表说明
表格内容
```

---

## 8.3 初版 font-family 建议

```css
:root {
  --font-sans:
    system-ui, -apple-system, BlinkMacSystemFont,
    "Segoe UI", "Microsoft YaHei", "PingFang SC",
    "Noto Sans CJK SC", sans-serif;

  --font-mono:
    "SFMono-Regular", Consolas, "Liberation Mono", Menlo, monospace;

  --font-pixel:
    var(--font-mono);
}
```

后续如果引入像素字体，只替换 `--font-pixel`，不要影响正文。

## 9. 边框、阴影、圆角规范

## 9.1 边框

Pixel Archive 不适合大面积玻璃拟态或过圆的现代卡片。

建议：

```txt
普通卡片：1px 浅边框
重点卡片：2px 边框
像素组件：直角或小圆角
首页终端：可使用双层边框或硬阴影
Notes 正文：不要使用重边框
```

---

## 9.2 阴影

| 区域 | 阴影策略 |
|---|---|
| Home Save Slot | 可用轻微硬阴影 |
| Project Card | 普通软阴影 + hover 轻微增强 |
| Lab Card | 可参考 Neo-Brutalist 硬阴影 |
| Notes 正文 | 尽量无阴影或极轻阴影 |
| Archive Entry | 可使用轻微发光边框 |
| 404 Panel | 可更明显 |

---

## 9.3 圆角

整体建议：

```txt
小圆角 / 直角为主
```

不要使用：

```txt
大圆角玻璃拟态
过多 pill-shaped 大胶囊
强烈 neumorphism
过度拟物
```

```txt

P1 NES.css 为辅
```


---

## 11.3 Notes

参考：

```txt
N1 Pico CSS
```

要求：

```txt

```

## 11.7 404

## 12. 动效规范

## 12.1 允许的动效

```txt
光标闪烁
卡片 hover 轻微位移
Save Slot hover 轻微发光
页面进入轻微淡入
主题切换平滑过渡
按钮 hover 边框高亮
链接 hover 颜色变化
```

---

## 12.2 禁止或避免的动效

```txt
大面积粒子背景
大幅视差滚动
长加载动画
频繁抖动
闪烁过快
扫描线覆盖正文
CRT 扭曲效果影响阅读
动画影响正文阅读
复杂角色动画阻塞首页开发
```

---

## 12.3 动效原则

```txt
小
慢
安静
可忽略
不打扰阅读
不增加明显维护成本
```

---

## 13. 暗色模式规范

暗色模式不是终端主题。

暗色模式应该是：

```txt
柔和深灰背景
暖白文字
低亮度紫绿点缀
卡片层级清晰
代码块可读
图片不过暗
边框仍可辨识
```

不要：

```txt
纯黑背景
荧光绿文字
全站 DOS 终端风
高对比闪烁效果
正文区扫描线
```

暗色模式应与浅色模式共享同一套布局和组件结构，只替换主题变量。

## 14. 图片与素材规范

图片路径和命名由 `04_内容与文件管理规范.md` 定义。本文只定义视觉使用方式。

---

## 14.1 首页素材

初版首页不依赖真实图片。

优先使用：

```txt
CSS 形状
SVG 占位
HTML 终端结构
像素方块装饰
```

后续可替换为：

```txt
像素头像
像素工作台
像素房间
像素地图入口
```

HomeScene 需要设计为可替换模块，不要把首页写死成某一张插画。

---

## 14.2 项目封面

项目封面建议：

```txt
真实截图 + 统一外框
无图项目使用 CSS/SVG 占位封面
不过度滤镜化
不破坏截图信息可读性
```

---

## 14.3 Lab 图片

Lab 可以更自由：

```txt
像素图
```

## 15. 空状态视觉

空状态应与 Pixel Archive 气质一致。

视觉上可以使用：

```txt
小像素边框
柔和提示卡片
低饱和状态图标
短 UI 文案
```

不要使用：

```txt
系统默认“暂无数据”
红色错误式空状态
大面积空白无提示
过度游戏化提示
```

具体文案由 `07_文案与内容风格规范.md` 定义。

---

## 16. 可访问性视觉要求

视觉实现不得牺牲基础可访问性。

至少保证：

```txt
颜色对比基本可读
hover 与 focus 状态可见
主题切换按钮可辨识
链接和按钮状态明确
正文行距舒适
代码块可读
暗色模式不刺眼
```

不要为了像素风降低可读性。

---

## 17. 视觉验收标准

开发完成后，用以下标准检查：

```txt
1. 首页打开时能感到 Pixel Archive 的独特气质
2. Projects 页面信息清楚，不像游戏菜单堆满屏
3. Notes 页面能舒服阅读长文
4. Archive 有存档点感觉，但不乱
5. Lab 比其他页面更自由
6. About 克制，不像简历模板
7. 404 有趣且符合 Pixel Archive 气质
8. 暗色模式不是纯黑荧光终端
9. 紫绿点缀克制，不高饱和
10. 像素元素不影响可读性
11. 第三方参考没有直接套模板感
12. 所有页面看起来属于同一个网站
13. 正文、公式、代码块不被像素风破坏
14. 首页有风格，但加载和维护成本不高
15. 移动端不崩，桌面端优先好看
```

更完整的阶段验收由 `09_验收与测试规范.md` 定义。

## 19. 最终原则

> Pixel Archive 的视觉不追求炫技，而追求长期稳定的个人识别感。  
> 它应该有像素风，但不幼稚；有复古终端感，但不模板化；有极简阅读体验，但不无聊。

## 20. Phase 20 动效实现边界

当前稳定动效仅包括：

```txt
READY 光标慢速 blink
状态像素慢速 pulse
首页 grid 在 18 秒内漂移 1–2px
卡片与链接的 1–2px hover 位移
主题切换时 180ms 的背景、边框、文字和阴影过渡
```

所有动画必须被以下媒体查询关闭或显著缩短：

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    scroll-behavior: auto !important;
    transition-duration: 0.01ms !important;
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
  }
}
```

侧边辅助保持低对比、小字号和桌面优先；Notes 目录在移动端转为静态块，About SideRail 在窄屏隐藏。

## 21. Phase 22 坐标网格母题

坐标网格代表档案纸、测量基线和像素坐标，不作为全页背景花纹。

稳定使用范围：

```txt
Home：主视觉保留最明显的 grid
Projects / Lab：程序化封面承担主要 grid 识别
Notes：仅在详情目录等辅助阅读区域使用极淡 grid
Archive：仅在时间轴记录轨道内使用极淡 grid
About：仅在 profile SideRail 内使用极淡 grid
404 / terminal empty state：允许使用更明确的二维坐标 grid
```

实现约束：

```txt
优先复用 .coordinate-grid 与 .coordinate-grid--strong
不得在中文正文、Markdown 主阅读区或整页背景重复铺设
暗色模式继续使用语义化 --color-grid token
纹理不得改变组件尺寸、交互区域或内容层级
```
