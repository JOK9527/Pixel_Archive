# 15_数据模型与内部接口规范

版本：v1.0  
项目：Pixel Archive  
位置：`docs/standards/15_数据模型与内部接口规范.md`  
用途：统一 Pixel Archive 的内容数据模型、frontmatter 字段、内部读取函数、slug / href 规则、组件数据输入、站点配置和 CSS Variables 等内部接口，避免字段名、状态值、路由和数据结构在后续开发中分裂。  

---

## 2. 内部接口总原则

Pixel Archive 当前没有后端 API，但存在很多“内部接口”。

这些内部接口包括：

```txt
Content Collections schema
frontmatter 字段
status / type 枚举
slug
href
src/lib/content 输出结构
组件 props
navigation.ts
site.ts
CSS Variables
图片路径
页面路由
```

核心原则：

> 字段名稳定、枚举值稳定、路由稳定、数据从内容层统一整理后再传给页面和组件。

## 3. 不允许随意变化的核心接口

以下属于稳定接口，不能随意改名：

```txt

Project.techStack
Project.featured
Note.type
Note.draft
Archive.type
Lab.status
Lab.type
slug
href
tags
category
title
description
date
```

以下路由也属于稳定接口：

```txt
/
/projects/
/projects/{slug}/
/notes/
/notes/{slug}/
/archive/
/lab/
/lab/{slug}/
/about/
```

如果需要修改，必须同步更新：

```txt

相关页面
相关组件
README
WORKLOG

```

## 4. Project 数据模型

Project 是项目展示条目。

## 4.1 Project Frontmatter 主字段

必填字段：

```ts
type ProjectFrontmatter = {
  title: string;
  description: string;
  type: "web" | "ai-tool" | "research" | "design" | "hardware" | "personal-site";
  status: "WIP" | "Done" | "Prototype" | "Paused";
  category: "AI / ML" | "Aerospace / Control" | "Tools / Web" | "Creative / Lab" | string;
  tags: string[];
  techStack: string[];
  featured: boolean;
};
```

可选字段：

```ts
type ProjectFrontmatterOptional = {
  startDate?: string;
  updatedDate?: string;
  endDate?: string | null;
  cover?: string;
  links?: {
    github?: string;
    demo?: string;
    docs?: string;
  };
  relatedNotes?: string[];
  relatedArchive?: string[];
};
```

---

## 4.2 Project 字段含义

| 字段 | 含义 | 备注 |
|---|---|---|
| `title` | 项目标题 | 显示用 |
| `description` | 1–2 句项目简介 | 不写营销文案 |
| `type` | 稳定项目类型 | 使用固定枚举 |
| `status` | 项目状态 | 使用固定枚举 |
| `category` | 主分类 | 数量保持克制 |
| `tags` | 标签 | 小写英文，短横线 |
| `techStack` | 技术栈 | 使用这个字段名，不改成 technologies |
| `featured` | 是否精选 | 首页和 Projects 重点展示 |
| `startDate` | 开始日期 | YYYY-MM-DD |
| `updatedDate` | 最近更新日期 | YYYY-MM-DD |
| `endDate` | 结束日期 | 可为空 |
| `cover` | 封面路径 | `/images/projects/...` |
| `links` | 外部链接 | github / demo / docs |
| `relatedNotes` | 关联 Notes slug | 使用 slug |
| `relatedArchive` | 关联 Archive slug | 使用 slug |

---

## 4.3 Project status 固定值

```txt
WIP
Done
Prototype
Paused
```

禁止新增近义状态：

```txt
Doing
In Progress
Finished
Completed
Experimental
Stopped
Archived
```

如果确实要新增，必须同时更新：

```txt
04_内容与文件管理规范.md
06_组件设计规范.md
15_数据模型与内部接口规范.md
ProjectStatus 组件
相关 Badge 映射
```

---

## 4.4 Project 输出模型

`src/lib/content/projects.ts` 建议统一输出：

```ts
type ProjectItem = {
  slug: string;
  href: string;
  title: string;
  description: string;
  type: "web" | "ai-tool" | "research" | "design" | "hardware" | "personal-site";
  status: "WIP" | "Done" | "Prototype" | "Paused";
  category: string;
  tags: string[];
  techStack: string[];
  featured: boolean;
  cover?: string;
  links?: {
    github?: string;
    demo?: string;
    docs?: string;
  };
  startDate?: string;
  updatedDate?: string;
  endDate?: string | null;
};
```

注意：

```txt
href 应在 content helper 或页面层统一生成。
ProjectCard 不应自己拼复杂路径。
`year.month` 由 `getYearMonth()` 从内容日期派生，不作为重复存储字段。
```

## 5. Note 数据模型

Note 是文章、日志和学习笔记的统一入口。

## 5.1 Note Frontmatter 主字段

必填字段：

```ts
type NoteFrontmatter = {
  title: string;
  type: "Article" | "Log" | "Note";
  category: string;
  tags: string[];
  date: string;
  draft: boolean;
};
```

可选字段：

```ts
type NoteFrontmatterOptional = {
  description?: string;
  updated?: string;
  cover?: string;
  relatedProjects?: string[];
  relatedArchive?: string[];
};
```

---

## 5.2 Note type 固定值

```txt
Article
Log
Note
```

含义：

| type | 含义 |
|---|---|
| Article | 结构完整、长期可读文章 |
| Log | 阶段记录、开发日志 |
| Note | 学习笔记、临时整理 |

禁止新增近义类型：

```txt
Blog
Post
Diary
Journal
Essay
Tutorial
```

除非后续重新设计内容体系。

---

## 5.3 Note 输出模型

```ts
type NoteItem = {
  slug: string;
  href: string;
  title: string;
  description?: string;
  type: "Article" | "Log" | "Note";
  category: string;
  tags: string[];
  date: string;
  updated?: string;
  cover?: string;
  draft: boolean;
};
```

`getPublishedNotes()` 应默认过滤：

```txt
draft: true
```

---

## 6. Archive 数据模型

Archive 是存档点，不是完整文章。

## 6.1 Archive Frontmatter 主字段

必填字段：

```ts
type ArchiveFrontmatter = {
  title: string;
  date: string;
  type: "save-point" | "milestone" | "project-log";
  description: string;
};
```

可选字段：

```ts
type ArchiveFrontmatterOptional = {
  relatedProject?: string;
  relatedNote?: string;
};
```

---

## 6.2 Archive type 固定值

```txt
save-point
milestone
project-log
```

显示时可以映射为：

```txt
SAVE POINT
MILESTONE
PROJECT LOG
```

但 frontmatter 中保持小写短横线。

---

## 6.3 Archive 输出模型

```ts
type ArchiveItem = {
  slug: string;
  title: string;
  date: string;
  type: "save-point" | "milestone" | "project-log";
  description: string;
  relatedProject?: string;
  relatedNote?: string;
  relatedHref?: string;
};
```

按年份 / 月份分组时建议输出：

```ts
type ArchiveMonthGroup = {
  month: string;
  entries: ArchiveItem[];
};

type ArchiveYearGroup = {
  year: string;
  months: ArchiveMonthGroup[];
};
```

## 8. Slug 规则

所有 slug 必须：

```txt
小写英文
短横线连接
不使用中文
不使用空格
不使用下划线
不使用特殊符号
上线后不随意修改
```

示例：

```txt
pixel-archive
scholar-lens-ai
servo-test-platform
2026-06-10-first-note
```

---

## 9. Href 生成规则

href 必须稳定。

推荐规则：

```ts
const projectHref = `/projects/${slug}/`;
const noteHref = `/notes/${slug}/`;
```

固定页面：

```txt
/
/projects/
/notes/
/archive/
/lab/
/about/
```

不要在多个组件里重复拼接复杂路径。  
建议由页面层或 `src/lib/content/*` 统一生成。

---

## 10. 图片路径接口

图片路径统一使用：

```txt
/images/...
```

示例：

```txt
/images/projects/pixel-archive/cover.png
/images/notes/2026-06-10-first-note/cover.png
/images/lab/pixel-ui-placeholder/cover.png
```

禁止：

```txt
C:/Users/...
./assets/...
![[image.png]]
```

图片路径规则主定义见：

```txt
04_内容与文件管理规范.md
```

## 11. src/lib/content 接口

建议目录：

```txt
src/lib/content/
├─ projects.ts
├─ notes.ts
├─ archive.ts
└─ lab.ts
```

## 11.1 projects.ts

推荐函数：

```ts
getAllProjects(): Promise<ProjectItem[]>
getFeaturedProjects(): Promise<ProjectItem[]>
getProjectBySlug(slug: string): Promise<ProjectItem | undefined>
```

规则：

```txt
负责读取 projects collection
负责排序
负责生成 href
负责筛选 featured
不负责渲染组件
```

---

## 11.2 notes.ts

推荐函数：

```ts
getAllNotes(): Promise<NoteItem[]>
getPublishedNotes(): Promise<NoteItem[]>
getNoteBySlug(slug: string): Promise<NoteItem | undefined>
getNotesByType(type: "Article" | "Log" | "Note"): Promise<NoteItem[]>
```

规则：

```txt
getPublishedNotes 默认过滤 draft
按 date 降序排序
不负责 Markdown 渲染样式
```

---

## 11.3 archive.ts

推荐函数：

```ts
getAllArchiveEntries(): Promise<ArchiveItem[]>
getArchiveGroups(): Promise<ArchiveYearGroup[]>
```

规则：

```txt
按 date 降序排序
可按年份 / 月份分组
不负责 ArchiveTimeline 样式
```

---

## 11.4 lab.ts

推荐函数：

```ts
getAllLabItems(): Promise<LabItem[]>
getLabBySlug(slug: string): Promise<LabItem | undefined>
```

规则：

```txt
负责读取 lab collection
负责生成 href
详情路由固定为 /lab/{slug}/
不负责 LabCard 视觉
```

---

## 12. 组件数据输入接口

组件不要直接接收原始 collection entry，除非该组件明确属于页面内部临时实现。

推荐：

```txt
页面 / lib/content 整理数据
业务组件接收简化后的 item
UI 组件接收 label / href / variant 等简单字段
```

示例：

```ts
<ProjectCard
  title={project.title}
  description={project.description}
  status={project.status}
  category={project.category}
  tags={project.tags}
  techStack={project.techStack}
  href={project.href}
  cover={project.cover}
  featured={project.featured}
/>
```

不推荐：

```ts
<ProjectCard entry={entry} />
```

除非 `ProjectCard` 明确知道并稳定依赖 `CollectionEntry<"projects">` 类型。

---

## 13. navigation.ts 接口

推荐结构：

```ts
export type NavItem = {
  label: string;
  href: string;
};

export const navigation: NavItem[] = [
  { label: "Home", href: "/" },
  { label: "Projects", href: "/projects/" },
  { label: "Notes", href: "/notes/" },
  { label: "Archive", href: "/archive/" },
  { label: "Lab", href: "/lab/" },
  { label: "About", href: "/about/" },
];
```

规则：

```txt
Header 读取 navigation.ts
不要在 Header 中重复写死导航
导航 label 保持稳定
导航 href 保持稳定
```

---

## 14. site.ts 接口

推荐结构：

```ts
export const site = {
  name: "Pixel Archive",
  description: "A personal archive for projects, notes, save points and experiments.",
  url: "",
  author: "",
  github: "",
  builtWith: "Astro",
};
```

规则：

```txt
站点名只在 site.ts 中集中维护
正式 URL 未确定时可以为空
不要在多个文件硬编码站点名和 URL
```

## 15. CSS Variables 接口

CSS Variables 也是内部接口。

基础变量建议：

```css
:root {
  --color-bg: ...;
  --color-surface: ...;
  --color-surface-muted: ...;
  --color-text: ...;
  --color-text-muted: ...;
  --color-border: ...;
  --color-border-strong: ...;
  --color-purple: ...;
  --color-purple-soft: ...;
  --color-green: ...;
  --color-green-soft: ...;
  --shadow-soft: ...;
  --shadow-pixel: ...;
  --radius-sm: ...;
  --radius-md: ...;
}
```

规则：

```txt
组件样式使用变量
不要大量硬编码颜色
不要随意改变量名
新增变量要说明用途
```

详细视觉定义见：

```txt
03_视觉风格规范.md
```

## 16. 状态映射规则

业务状态到 UI variant 的映射应放在业务组件中。

示例：

```txt
ProjectStatus 负责把 WIP 映射成 warning / purple
Badge 只负责显示 label + variant
```

不要让基础 UI 组件理解业务枚举。

推荐边界：

```txt
ProjectStatus 知道 Project.status
NoteTypeBadge 知道 Note.type
LabTypeBadge 知道 Lab.type
Badge 不知道这些业务类型
```

---

## 17. 日期格式

内容日期使用：

```txt
YYYY-MM-DD
```

展示时可以格式化为：

```txt
2026.06.10
June 10, 2026
2026-06-10
```

但 frontmatter 内保持：

```txt
YYYY-MM-DD
```

不要混用：

```txt
2026/6/10
2026年6月10日
10-06-2026
```

---

## 18. 关联字段规则

关联字段必须使用 slug。

推荐：

```yaml
relatedNotes:
  - "2026-06-10-first-note"

relatedProjects:
  - "pixel-archive"

relatedProject: "pixel-archive"
relatedNote: "2026-06-10-first-note"
```

不推荐：

```yaml
relatedProjects:
  - "Pixel Archive"
```

原因：

```txt
标题会变，slug 应长期稳定。
```

---

## 19. 内部接口变更流程

如果必须修改字段名、枚举、路由或 helper 输出结构，应执行：

```txt
1. 说明为什么必须修改
2. 列出影响范围
3. 修改 schema
4. 修改内容文件
5. 修改 lib/content
6. 修改页面
7. 修改组件 props
8. 修改对应规范
9. 执行 npm run build
10. 更新 WORKLOG
```

不得只修改一处。

## 20. 常见错误

## 20.1 字段名分裂

错误：

```txt
frontmatter: techStack
component: technologies
page: stack
```

正确：

```txt
统一使用 techStack
```

---

## 20.2 状态值分裂

错误：

```txt
WIP
Doing
In Progress
```

正确：

```txt
统一使用 WIP
```

---

## 20.3 组件自己拼路径

错误：

```txt
ProjectCard 内部根据 title 拼 /projects/...
```

正确：

```txt
页面或 lib/content 传入 href
```

---

## 20.4 UI 组件理解业务

错误：

```txt
Badge 内部判断 Project status
```

正确：

```txt
ProjectStatus 映射 status
Badge 只显示 label 和 variant
```

## 22. 验收标准

检查内部接口时，至少确认：

```txt
1. frontmatter 字段与 schema 一致
2. schema 字段与 lib/content 输出一致
3. lib/content 输出与组件 props 一致
4. Project status 使用固定枚举
5. Note type 使用固定枚举
6. Archive type 使用固定枚举
7. Lab type / status 使用固定枚举
8. slug 规则统一
9. href 生成统一
10. Header 导航来自 navigation.ts
11. site 信息来自 site.ts
12. 图片路径使用 /images/...
13. CSS Variables 没有随意改名
14. 构建通过
15. WORKLOG 记录接口变更
```

## 24. 最终原则

> Pixel Archive 可以没有后端 API，但不能没有内部接口意识。  
> frontmatter、schema、helper 输出、组件 props、路由、slug 和 CSS Variables 都是契约；契约稳定，项目才不会在后续开发中分裂。
