# 17_公开信息与隐私边界规范

版本：v1.0  
项目：Pixel Archive  
位置：`docs/standards/17_公开信息与隐私边界规范.md`  
用途：规范 Pixel Archive 作为公开个人网站时，哪些信息可以公开、哪些信息需要谨慎、哪些信息禁止公开，以及上线前如何检查图片、内容、代码、链接和项目截图中的隐私风险。  

---

## 1. 本文档的作用

本文档负责回答：

```txt
1. Pixel Archive 能公开哪些个人信息？
2. About 页面应该写到什么程度？
3. 哪些项目内容可以放，哪些需要处理后再放？
4. 截图、图片、日志中哪些信息需要打码？
5. 是否可以公开邮箱、学校、专业、GitHub 等信息？
6. API Key、本地路径、账号、论文资料等如何处理？
7. 上线前如何做隐私检查？
```

本文档不负责法律合规意见，只作为个人网站公开信息边界的项目规范。

---

## 2. 总原则

Pixel Archive 是公开个人网站。  
公开内容应遵循：

```txt
必要
克制
可控
不暴露敏感信息
不公开无关隐私
不公开未准备好的个人材料
不公开他人信息
不公开 API Key / token / 本地路径
```

核心原则：

> 能少写就少写，能用项目说话就不用个人信息堆叠，能用链接替代就不要公开过多隐私细节。

---

## 3. 公开信息分级

信息分为四级：

```txt
A 级：可公开
B 级：谨慎公开
C 级：默认不公开
D 级：禁止公开
```

---

## 4. A 级：可公开信息

通常可以公开：

```txt
项目名称
项目简介
技术栈
项目截图中不含敏感信息的部分
公开 GitHub 仓库链接
公开 Demo 链接
公开学习笔记
公开文章
公开视觉实验
公开 Minecraft 建筑展示
公开已经处理过的图片素材
公开邮箱别名或专门联系邮箱
```

注意：

```txt
“可公开”不代表必须公开。
```

---

## 5. B 级：谨慎公开信息

谨慎公开：

```txt
真实姓名
学校
专业
年级
所在地
个人邮箱
微信 / QQ / 手机号
毕业论文全文
实验数据
课程作业
答辩 PPT
导师 / 同学 / 合作者姓名
项目中的真实路径
真实设备序列号
截图中的文件目录
```

公开前应判断：

```txt
是否必要
是否会影响隐私
是否包含他人信息
是否包含未公开资料
是否可以部分打码
是否可以用概述替代
```

---

## 6. C 级：默认不公开信息

默认不公开：

```txt
身份证件
学生证
具体住址
手机号
微信号
QQ号
私人聊天截图
未授权他人照片
未授权他人姓名和联系方式
未公开实验原始数据
未公开论文评阅意见
学校内部文档
导师批注意见原文
本地完整文件路径
命令行中暴露的用户名路径
```

除非用户明确决定公开，否则不放入网站。

---

## 7. D 级：禁止公开信息

禁止公开：

```txt
API Key
Access Token
Secret Key
Cookie
Session
.env
.env.local
数据库密码
部署密钥
GitHub token
Cloudflare token
OpenAI / DeepSeek / 智谱 / 其他模型 API Key
身份证号
银行卡号
精确住址
他人隐私信息
未授权版权资源
```

一旦发现，应立即删除并检查 Git 历史是否需要处理。

---

## 8. About 页面公开边界

About 页面应极短。

推荐公开：

```txt
这是一个个人档案网站
存放项目、笔记、阶段存档和实验内容
少量方向描述
GitHub 链接
可选联系邮箱
```

不建议公开：

```txt
完整履历
学校详细经历
班级
学号
手机号
微信
QQ
详细住址
大量个人照片
完整技能简历
```

推荐写法：

```txt
Pixel Archive 是一个个人档案馆，用来存放项目、笔记、阶段存档和一些实验性内容。
```

---

## 9. 项目页面公开边界

Projects 可以公开：

```txt
项目目标
技术栈
阶段状态
架构概述
公开截图
公开链接
开发总结
```

需要谨慎：

```txt
本地路径
私有仓库地址
API 配置
.env
实验原始数据
学校内部材料
未公开论文内容
未打码控制台截图
```

禁止：

```txt
API Key
私有 token
真实密码
未授权数据
他人隐私
```

---

## 10. Notes 公开边界

Notes 可以公开：

```txt
学习笔记
公开论文阅读笔记
工具使用记录
项目开发总结
技术概念整理
公开资料的二次整理
```

谨慎公开：

```txt
课程作业原文
论文初稿
导师批注
未公开实验数据
答辩问题原文
学校内部模板
他人发来的资料
```

建议：

```txt
把敏感材料改写成个人总结
不要直接贴私密原文
不要公开未授权文件
```

---

## 11. Archive 公开边界

Archive 是阶段存档。

可以公开：

```txt
项目启动
阶段完成
网站上线
某个公开项目 milestone
学习阶段总结
```

不建议公开：

```txt
非常私人日记
情绪化长文
具体隐私事件
他人相关信息
学校内部节点
```

Archive 应保持：

```txt
短
客观
像存档点
不暴露过多私人细节
```

---

## 12. Lab 公开边界

Lab 可以公开：

```txt
像素 UI 实验
AI 图像实验
Minecraft 建筑
小型交互 demo
视觉组件测试
```

谨慎公开：

```txt
使用他人素材的实验
带有人脸的生成图
包含版权风险的图像
未处理的游戏截图
包含个人账号信息的截图
```

---

## 13. 图片和截图检查

上传图片前必须检查：

```txt
是否包含 API Key
是否包含账号名
是否包含邮箱
是否包含微信 / QQ
是否包含本地路径
是否包含浏览器书签隐私
是否包含私有仓库名
是否包含未公开文件名
是否包含他人聊天内容
是否包含二维码
是否包含定位信息
是否包含人脸或他人照片
```

需要时处理：

```txt
裁剪
打码
重截
使用占位图
使用示意图
重新导出干净截图
```

---

## 14. 本地路径处理

禁止公开完整本地路径，例如：

```txt
C:/Users/用户名/Desktop/...
D:/private/project/...
E:/projects/...
```

如果需要说明目录结构，使用抽象路径：

```txt
project-root/
src/content/
public/images/
```

或者：

```txt
本地项目目录
项目根目录
```

---

## 15. API 配置处理

禁止公开：

```txt
API Key
Base URL 中包含 token 的链接
.env
请求 header
Authorization
Cookie
Secret
```

配置示例应写成：

```env
OPENAI_API_KEY=your_api_key_here
PUBLIC_SITE_URL=https://example.com
```

不要写真实值。

---

## 16. GitHub 链接规则

可以公开：

```txt
公开仓库
稳定 Demo
公开文档
```

谨慎公开：

```txt
私有仓库
还没清理历史的仓库
含有 .env 历史的仓库
含有本地路径或私密截图的仓库
```

公开前检查：

```txt
git status
.gitignore
仓库历史是否有敏感信息
README 是否暴露隐私
issues / discussions 是否有隐私
```

---

## 17. 邮箱和联系方式

推荐：

```txt
使用专门联系邮箱
使用 GitHub profile
使用 mailto 链接前谨慎考虑垃圾邮件
```

不建议：

```txt
公开私人常用邮箱
公开手机号
公开微信号
公开 QQ 号
```

可选方案：

```txt
只放 GitHub
只放邮箱图片
只放混淆邮箱
后续再加联系表单
```

---

## 18. 学业和论文资料边界

由于 Pixel Archive 可能涉及论文、毕设、实验和学习内容，需特别注意：

谨慎公开：

```txt
毕业论文全文
答辩 PPT
导师意见
学校模板
未公开实验数据
原始 CSV / MATLAB 数据
课程任务书
评阅表
同学或老师姓名
```

可以公开：

```txt
抽象后的项目总结
已经脱敏的系统架构图
不含隐私的实验结果图
通用技术笔记
自己整理的公开学习记录
```

建议：

```txt
论文相关内容先写成项目总结，不直接上传完整原始文档。
```

---

## 19. 他人信息处理

不得公开：

```txt
他人姓名
联系方式
照片
聊天记录
作业内容
批注
邮箱
账号
头像
未经授权的作品
```

如果必须提到合作者：

```txt
先确认对方是否允许
只写公开名称或 GitHub
避免暴露私人联系方式
```

---

## 20. 版权与素材

谨慎使用：

```txt
第三方插画
图标包
字体文件
游戏素材
网站模板截图
论文图表
他人照片
AI 生成的疑似版权角色
```

推荐：

```txt
自制 SVG
自制像素图
公开许可资源
自己生成并确认可用的素材
简单 CSS 图形
```

禁止：

```txt
未经许可分发字体文件
复制商用图标资源
直接搬运他人设计稿
```

## 21. 公开前检查清单

上线前检查：

```txt
1. 没有 .env
2. 没有 API Key
3. 没有 token
4. 没有 Cookie
5. 没有本地绝对路径
6. 没有手机号
7. 没有微信 / QQ
8. 没有身份证 / 学号
9. 没有未授权他人信息
10. 没有私密聊天截图
11. 没有未打码控制台截图
12. 没有私有仓库敏感链接
13. 没有未授权论文 / 学校资料
14. 图片中没有敏感信息
15. README 没有隐私信息
16. WORKLOG 没有敏感路径或 token
17. public/images 中没有未处理图片
18. content 中没有私密内容
19. Git 历史没有明显敏感提交
20. 部署环境变量未暴露到前端
```

## 22. Agent 处理隐私规则

Agent 在生成或修改内容时不得编造或暴露：

```txt
真实姓名
学校
手机号
微信
邮箱
家庭住址
API Key
token
账号密码
他人信息
未授权资料
```

如果用户提供了敏感材料，Agent 应提醒：

```txt
这部分不建议直接公开，可以改写成脱敏总结。
```

Agent 生成占位内容时应使用：

```txt
example.com
your-email@example.com
your_api_key_here
project-root/
```

不要使用看起来真实的敏感值。

---

## 23. 隐私风险分级处理

发现敏感信息后按级别处理：

## 23.1 低风险

例如：

```txt
普通本地路径
过多个人介绍
未必要公开的学校信息
```

处理：

```txt
改写
删除
替换为泛化描述
```

## 23.2 中风险

例如：

```txt
邮箱
真实姓名
学校内部资料
未打码截图
```

处理：

```txt
询问是否公开
默认删除或打码
更新公开版本
```

## 23.3 高风险

例如：

```txt
API Key
token
密码
身份证号
手机号
他人隐私
```

处理：

```txt
立即删除
不要提交
如已提交，提示需要清理 Git 历史并轮换密钥
```

---

## 24. 与内容风格的关系

隐私边界优先于文案风格。

例如：

```txt
About 页面可以写得有风格
但不能为了风格公开过多私人信息

Archive 可以像存档点
但不能像日记一样公开隐私事件

Project 可以展示技术能力
但不能公开 API 配置和私有数据
```


---

## 26. 最终原则

> Pixel Archive 是公开档案馆，不是私人硬盘镜像。  
> 公开内容应保留项目价值和个人风格，但删除敏感信息、私人路径、密钥、他人信息和未授权材料。能脱敏就脱敏，能概述就概述，不确定就先不公开。
