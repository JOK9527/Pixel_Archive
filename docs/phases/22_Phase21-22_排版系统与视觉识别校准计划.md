## 1. 与既有阶段的关系

已有阶段关系如下：

```txt
Phase 0–9：结构基线、页面路由、内容集合、第一阶段验收
Phase 10：视觉系统再校准、全站配色与视觉层级

Phase 16–20：分类索引、Lab 详情、Archive 管理、语义化像素封面、侧边辅助与克制动效
```

本文件规划的 Phase 21–22 不重复上述任务。

它解决的是 Phase 16–20 之后仍然会暴露出的两个系统性问题：

```txt
Phase 21：排版栅格、间距系统、卡片节奏、正文 measure 的统一

```

也就是说：

```txt
Phase 16–20 解决“信息怎么管理、卡片怎么打开、Archive 怎么筛选、封面怎么语义化”。
Phase 21–22 解决“这些东西放到页面里之后，是否像同一个设计系统排出来”。
```

## 2. 当前问题复盘

根据当前页面截图与人工反馈，问题集中在以下方面。

### 2.1 间距系统不统一

现象：

```txt
hero → 主内容的过渡间距各页不同
Home / Notes 的 gap 明显大于 Archive / Lab
卡片内 padding、列表 gap、section gap 各自为政
About metadata table 偏松
Sidebar 标签和筛选项偏紧
```

问题：

```txt
看不出统一的 spacing baseline
组件像分别写出来，而不是从同一套 spacing scale 派生
```

---

### 2.2 Sidebar 又瘦又挤

现象：

```txt
Projects / Notes / Archive / Lab 左侧 Index 面板窄
分类、状态、year.month、技术栈等信息挤在竖条里
与右侧宽主内容比例失调
section 分隔不够明确
```

问题：

```txt
左侧索引本来用于填补空白和增强管理，但现在像“小配件”
没有成为主布局的一部分
```

---

### 2.3 卡片内部垂直节奏紊乱

现象：

```txt
metadata / 标题 / 摘要 / 标签 / 日期 / action 的间距没有统一规则
Notes metadata 行用斜杠塞入过多信息
Projects 预览图和正文之间 gap 偏大
Lab 描述、标签、日期、OPEN 的层级差不明显
Archive 卡片与时间轴节点对齐关系不精确
```

问题：

```txt
卡片可用，但不像同一套组件系统
交互入口与元信息的视觉权重不清晰
```

---

### 2.4 正文 measure 没有设计

现象：

```txt
Notes 列表卡片很宽，但文字列宽不稳定
Lab 卡片中文描述换行显得随机
About 正文短而飘
Notes 详情页长中文标题过大，多行后压迫感强
```

问题：

```txt
卡片宽度被定义了，但文字列宽没有被定义
阅读页和列表页的文本节奏没有分别处理
```

---

### 2.5 中英混排规则不统一

现象：

```txt
Home 全英文
Projects / Notes / Archive / Lab 中文副标题与中英内容混杂
LOG / NOTE / ARTICLE 等英文标签和中文内容并列
部分页面状态英文，部分说明中文
```

问题：

```txt
中英混排不是问题，随机混排才是问题
当前缺少语言角色分工
```

---

### 2.6 首页 grid 视觉母题没有跨页延续

现象：

```txt
首页 hero grid 是最有识别度的视觉元素
Projects / Notes / Archive / Lab 中 grid 只局部出现或几乎消失
```

问题：

```txt
跨页视觉识别不够连续
首页像一个独立视觉系统，子页面像另一个系统
```

---

### 2.7 像素感仍然偏弱

现象：

```txt
像素感主要来自角标、小色块、chart icon、局部 grid
大标题仍然偏普通粗体无衬线
页面整体像“带像素点缀的极简站”，而不是“Pixel Archive”
```

问题：

```txt
像素元素没有形成足够强的识别策略
但又不能为了像素化牺牲中文正文阅读
```

---

### 2.8 马赛克 / grid 图形规律不清晰

现象：

```txt
Projects / Home / Lab 中反复出现马赛克 grid 或像素块图形
难以看出图形与内容的对应关系
美观性和信息含义都不稳定
```

问题：

```txt
如果是纯装饰，它占比过大
如果是信息图形，它缺少可理解规则
```

结论：

```txt
不适合长期手工制作每个项目的视觉资产
应转向元数据驱动的程序化像素封面系统
```

## 3. Phase 21：排版栅格、间距系统与内容节奏校准

### 3.1 阶段目标

Phase 21 的目标不是新增功能，而是建立全站稳定的排版基础。

目标：

```txt
统一 spacing scale
统一页面栅格
统一 hero → content 过渡
统一 sidebar 与 main content 比例
统一卡片内部垂直节奏
统一正文 measure
修正 Archive 时间线结构对齐
修正 Notes 详情页长标题阅读压力
```

本阶段做完后，页面应该从：

```txt
“各组件分别摆得还可以”
```

变成：

```txt
“同一套 layout system 排出来的页面”
```

---

### 3.2 不做内容

本阶段不做：

```txt
不新增真实内容
不新增大功能
不调整 Content Collections schema，除非只是补充可选 display 字段
不继续大规模调色
不重新设计首页
不引入新依赖
不做复杂动效
不重写组件结构
```

---

### 3.3 建立 spacing scale

建议在全局 CSS 中建立 spacing token。

建议基准：

```css
:root {
  --space-0: 0;
  --space-1: 0.25rem;  /* 4px */
  --space-2: 0.5rem;   /* 8px */
  --space-3: 0.75rem;  /* 12px */
  --space-4: 1rem;     /* 16px */
  --space-5: 1.5rem;   /* 24px */
  --space-6: 2rem;     /* 32px */
  --space-7: 3rem;     /* 48px */
  --space-8: 4rem;     /* 64px */
  --space-9: 6rem;     /* 96px */
}
```

要求：

```txt
所有 section gap、card padding、sidebar padding、列表间距优先使用 token
不得继续散落 17px / 23px / 37px 等无规律值
特殊值必须注释说明原因
```

---

### 3.4 建立页面级 layout grid

建议统一页面结构：

```txt
PageShell
├─ PageHeader
└─ ContentGrid
   ├─ LeftIndex / SideRail
   └─ MainContent
```

建议桌面端参考：

```txt
页面最大宽度：1120–1240px
左侧索引：220–260px
主内容：760–900px
列间距：40–56px
右侧空白：自然留白，不强行填满
```

建议移动端：

```txt
LeftIndex 折叠到内容上方
或显示为横向 chip group
主内容单列
无横向滚动
```

---

### 3.5 统一 hero → content 过渡

定义页面头部到正文的统一间距。

建议：

```txt
PageHeader 底部到 ContentGrid 顶部：64px 桌面 / 40px 移动端
Home 可特殊处理，但必须注释为 hero layout exception
Notes / Projects / Archive / Lab / About 不应各自随机设置
```

如果页面内容少，可用背景、侧边索引或辅助说明填补，不靠过大空白撑开。

---

### 3.6 Sidebar 系统校准

#### 3.6.1 宽度

建议：

```txt
桌面端 sidebar 宽度：220–260px
低于 220px 时不放复杂筛选
内容少于 10 条时可显示 summary index
```

#### 3.6.2 内部结构

建议分区：

```txt
INDEX HEADER
All records
CATEGORY / TYPE
STATUS
YEAR.MONTH
TAGS / TECH
Footer note
```

每个分区必须有：

```txt
明确 section title
分区间距
弱分割线或 pixel divider
计数右对齐
active 状态
```

#### 3.6.3 使用边界

Sidebar 不应该变成后台筛选器。

它的气质应是：

```txt
archive index
catalog card
drawer label
```

不是：

```txt
admin filter panel
heavy control sidebar
```

---

### 3.7 卡片内部节奏模板

每类卡片应有固定内部节奏。

#### 3.7.1 ProjectCard

建议结构：

```txt
Cover / visual block
Meta row
Title
Description
Tags / tech
Action row
```

节奏：

```txt
cover → meta：24px
meta → title：12px
title → description：8–12px
description → tags：16px
tags → action：20–24px
```

#### 3.7.2 NoteItem

Notes metadata 不要全部塞一行。

建议拆成：

```txt
第一行：type badge + category + date
第二行：updated / reading state / optional metadata
Title
Summary
Tags
OPEN NOTE →
```

如果 metadata 太短，可合并；太长必须换行。

`OPEN NOTE →` 应比 metadata 更明显。

#### 3.7.3 ArchiveItem

Archive 卡片必须与时间轴节点有精确关系。

要求：

```txt
时间轴节点与卡片 top edge 对齐
日期归属时间轴，不漂浮在两者之间
事件类型 label 归属卡片右上角
卡片间距可根据年份 / 月份分组略有变化
```

#### 3.7.4 LabCard

建议结构：

```txt
Experiment visual
Meta row
Title
Description
Tags
Date
OPEN EXPERIMENT →
```

短描述和长描述都要稳定。

如果卡片等高：

```txt
Action row 固定在底部
描述区允许自然伸缩
标签和日期靠近 action
```

---

### 3.8 正文 measure 规则

建议定义：

```css
:root {
  --measure-narrow: 38rem;
  --measure-reading: 46rem;
  --measure-wide: 58rem;
}
```

使用规则：

```txt
Notes 详情正文：--measure-reading
About 正文段落：--measure-reading 或 --measure-narrow
卡片描述：22–34rem 内控制
Hero 副标题：不超过 32rem
中文长标题：单独设置 max-width 和 line-height
```

Notes 详情页长中文标题建议：

```txt
桌面端 max font-size 不超过 4.5rem
窄屏下 2.6–3.2rem
line-height 0.95–1.08 之间按视觉测试
标题 max-width 不宜过窄，避免 2–3 字一行
```

---

#### Home

修正 hero 左右视觉重量
标题侧和菜单侧建立共同中轴或基线
确认 Save Slots 与 Hero 面板宽度关系
避免左空右密的拼贴感
```

#### Projects

```

#### Notes

```txt

OPEN NOTE 更像行动入口

```

#### Archive

```txt

时间线不再只是装饰虚线
```

#### Lab

```txt
补 OPEN EXPERIMENT 入口后统一卡片底部节奏
实验封面与卡片边框共享网格基准
```

```txt
左侧 profile card 与右侧正文建立更紧密关系
metadata table 收紧垂直间距
底部 2×2 卡片减少空置感或降级为更轻量入口
```

## 4. Phase 22：语言系统、跨页视觉母题与元数据驱动像素封面收口

### 4.1 阶段目标

Phase 22 的目标是增强 Pixel Archive 的识别度，同时避免维护成本失控。

目标：

```txt
建立中英混排规则
延续首页 grid / 坐标纸视觉母题到子页面
加强像素字体与 UI 标签策略
将装饰性马赛克升级为元数据驱动像素封面
定义静置动效边界
避免手工封面资产成为长期负担
```

本阶段不是继续堆装饰，而是把已有像素元素变成可维护的视觉系统。

---

### 4.2 中英混排规则

建议建立语言角色分工。

#### 4.2.1 英文用于系统层

包括：

```txt
UI 标签
系统状态
坐标标记
操作入口
内容类型 badge
技术标签
slug / category code
```

示例：

```txt
PROJECT INDEX
ARCHIVE DRAWER / 02
OPEN NOTE →
OPEN EXPERIMENT →
SAVE POINT
WIP / DONE / LOG / NOTE / ARTICLE
```

#### 4.2.2 中文用于内容层

包括：

```txt
页面副标题
项目简介
笔记标题
摘要
正文
Archive 描述
About 说明
```

示例：

```txt
一些正在完成、已经完成和暂时搁置的项目。
文章、学习记录和开发日志。
个人档案站的公开边界。
```

#### 4.2.3 技术标签保留英文小写

示例：

```txt
#astro
#typescript
#mdx
#css
#content-collections
#responsive
```

#### 4.2.4 不允许随机混排

避免：

```txt
同一层级中有些标签中文、有些标签英文
同一类状态一会儿 WIP，一会儿 进行中
同一操作一会儿 OPEN，一会儿 查看详情
```

---

### 4.3 跨页 grid / 坐标纸视觉母题

首页 hero grid 是当前最有辨识度的视觉元素之一。

Phase 22 不要求每页都大面积铺 grid，但应抽象成跨页母题。

建议：

```txt
Home：hero 区保留最明显 grid
Projects：项目封面区使用 grid
Notes：详情页 TOC / code / callout 可使用极淡 grid
Archive：时间轴背景局部使用 grid / coordinate tick
Lab：实验卡封面使用明显 grid
About：profile card 内使用极淡 grid
404：lost coordinate / save-file grid 可更明显
```

原则：

```txt
grid 是档案纸 / 坐标纸 / 像素基准线
不是背景花纹
不得影响正文阅读
不得每页满屏铺同一套纹理
```

---

### 4.4 像素字体与标题策略

当前像素感不足，但不应全站使用强像素字体。

建议策略：

```txt
中文正文：保持高可读字体
中文标题：可保持现有无衬线，但控制字号和行高
英文品牌 / UI 标签：增强像素感或 blocky 感
系统标识：使用 monospace / pixel-style 字体
```

可尝试范围：

```txt
Pixel Archive 品牌字样
PA-00 / INDEX
ARCHIVE ONLINE
SAVE SLOT
OPEN NOTE →
OPEN EXPERIMENT →
LOG / NOTE / ARTICLE
```

不建议：

```txt
中文正文像素字体化
长文标题强像素化
所有按钮和正文都使用游戏字体
```

---

### 4.5 元数据驱动像素封面系统

#### 4.5.1 基本判断

不建议长期采用“每个项目单独制作视觉资产”。

原因：

```txt
维护成本高
新增项目会增加设计负担
不同资产难以保持统一质量
项目数量增加后容易风格混乱
```

推荐方向：

```txt
固定模板 + metadata 自动生成差异
```

也就是：

```txt
不是手工封面资产系统
而是元数据驱动的程序化像素封面系统
```

---

#### 4.5.2 建议模板

控制在 4 套以内：

```txt
Save Chip：适合 Projects
Terminal Strip：适合 Notes / 技术记录
Timeline Fragment：适合 Archive
Experiment Grid：适合 Lab
```

---

#### 4.5.3 数据字段建议

如后续需要扩展，可加入可选字段：

```ts
coverVariant?: 'save-chip' | 'terminal-strip' | 'timeline-fragment' | 'experiment-grid';
coverSeed?: string;
accent?: 'green' | 'blue' | 'amber' | 'violet' | 'red';
visualDensity?: 'low' | 'medium' | 'high';
```

注意：

```txt
字段应可选
没有字段时使用默认模板
不得为了封面系统破坏现有内容 schema
不得要求每篇内容手动配置复杂视觉参数
```

---

#### 4.5.4 映射规则示例

Projects：

```txt
type 决定主色
status 决定角标
tech 数量决定块数
year.month 决定底部短线节奏
featured 决定是否使用高密度模板
```

Notes：

```txt
category 决定主色
type 决定 badge
字数 / 阅读时间决定条形长度
tags 数量决定点阵密度
```

Archive：

```txt
event type 决定图形类型
month 决定节点位置 / 色点
milestone 使用更明显的时间片段
project-log 使用低密度线性模板
```

Lab：

```txt
experiment type 决定模板变体
status 决定角标
视觉实验可使用更高颜色密度
UI / component 实验使用更规整网格
```

---

#### 4.5.5 美观约束

元数据驱动不等于随机。

必须遵守：

```txt
最多 4–5 种颜色同时出现
同一模板内部保持固定网格
色块密度有上限
卡片内 cover 与正文使用同一内边距基准
封面不能比内容更抢眼
```

如果某图形无法解释含义：

```txt
要么降级为低存在感装饰
要么移除
```

---

### 4.6 静置动效边界

动效不只发生在 hover。

允许轻量静置动效：

```txt
READY 方块慢速 blink
ARCHIVE ONLINE 状态点轻微 pulse
首页 grid 极慢漂移 1–2px
像素角标偶尔轻微变色
状态栏短线极低频 scan
```

必须遵守：

```txt
周期慢
幅度小
不影响阅读
不影响性能
prefers-reduced-motion 下关闭
同屏最多 1–2 个轻微静置动效
```

禁止：

```txt
大面积闪烁
复杂滚动动画
霓虹发光
游戏 loading 式动效
强烈视差
需要复杂动画库的效果
```
