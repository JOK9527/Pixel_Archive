## 0. 项目源信息复读

在制定 Phase 16–20 计划前，必须先复读 Pixel Archive 的项目源信息，避免后续开发偏离既定方向。

### 0.1 项目定位

Pixel Archive 是一个个人网站项目，但不是传统简历页，也不是普通博客模板。

项目核心定位是：

```txt
Pixel Archive 是一个以极简阅读体验为底层、以像素组件为识别符号、以复古终端 / 存档隐喻为氛围的个人档案馆。
```

长期用于存放和展示：

```txt
项目作品
技术笔记
开发日志
阶段性存档
视觉 / 像素 / Minecraft / 图像实验内容
```

### 0.2 核心设计理念

```txt
极简做骨架
像素做识别
复古做氛围
内容可长期沉淀
结构可持续扩展
```

理解方式：

```txt
极简负责：阅读、留白、信息层级、长期维护
像素负责：按钮、边框、标签、首页识别、404、Lab
复古负责：archive、save slot、terminal、save point 等隐喻
```

网站可以有个人风格，但不能牺牲阅读体验和信息清晰度。

### 0.3 页面范围与页面职责

导航固定为：

```txt
Home / Projects / Notes / Archive / Lab / About
```

同时包含：

```txt
404 页面
```

页面职责：

```txt
Home：私人像素档案终端入口，不是个人简介页
Projects：项目与作品展示
Notes：文章、学习笔记、开发日志
Archive：重要阶段和项目存档点
Lab：视觉、像素、Minecraft、图像、小工具等实验内容
About：极简个人入口与联系方式，不做正式简历页
404：lost archive / save file not found 概念
```

## 1. 当前问题归纳

根据当前页面反馈，Phase 16–20 需要解决的问题如下。

### 1.1 Projects / Notes 缺少长期分类管理

当前 Projects / Notes 已可展示内容，但当内容增长后，仅靠列表和标签会难以管理。

需要建立：

```txt
分类
标签
年月索引
type / status / tech 等维度
轻量筛选或索引入口
```

用户倾向：

```txt
year 可以细化为 year.month
左侧空白可用于索引，既解决管理问题，也缓解大面积空白
```

### 1.2 Lab 卡片没有交互入口

当前 Lab 页面显示实验卡片，但卡片无法点击，也没有 `OPEN EXPERIMENT` 一类入口。

问题：

```txt
卡片视觉上像内容入口
实际不能打开
容易让用户误以为页面坏了
```

需要补齐：

```txt
/lab/[slug]
LabCard 可点击
Lab 详情页最小可用结构
```

### 1.3 Archive 时间轴长期管理不足

Archive 当前是单条时间轴，适合展示，但不适合长期管理大量节点。

需要补齐：

```txt
年份 / 月份索引
类型筛选
保留时间线气质
避免变成后台列表
```

### 1.4 侧边空白尚未系统化利用

多页面左右留白较多，当前主要为空间感服务。

后续应根据页面类型放入低干扰辅助信息：

```txt
年月索引
分类索引
On this page
当前 section 标识
archive ruler
阅读进度
```

### 1.5 像素图形重复且缺少语义

当前 Home / Projects / Lab 等页面存在类似像素封面图形，但主要是装饰。

问题：

```txt
重复度高
缺少实际含义
不同内容之间差异不明显
美观性容易随着内容增多而下降
```

应升级为：

```txt
语义化像素封面系统
由内容元数据决定图形模板、颜色、块数、条形长度和状态点
```

### 1.6 动效缺失但不能过度

当前页面较静态。

后续可以加入克制动效：

```txt
hover 微反馈
静置状态轻微 blink / pulse / scan
页面进入轻微过渡
主题切换平滑
```

但不得引入复杂动画库，也不得影响阅读。

## 2. 总体执行原则

### 2.1 不做大改版

Phase 16–20 不推翻现有页面结构与视觉系统。

允许：

```txt
新增轻量索引
新增详情页
新增筛选维度
新增语义化封面组件
新增微动效
```

不允许：

```txt
重做首页
重做全站布局
再次大规模调色
引入复杂视觉资产
引入外部 UI 框架
引入动画库
引入数据库或 CMS
```

### 2.2 信息系统优先

后续重点不是让页面“更花”，而是让网站更可维护。

优先级：

```txt
内容可管理
入口可理解
分类可扩展
图形有语义
空白有用途
动效有节制
```

### 2.3 静态优先

所有筛选、索引与详情页都应尽量基于 Astro 静态生成。

允许少量原生客户端 JS 用于：

```txt
前端筛选
筛选状态切换
小型动效开关
```

但不得引入第三方状态管理或搜索库。

### 2.4 先数据，后组件，再页面

每阶段都应遵循：

```txt
1. 确认 frontmatter 字段
2. 确认 helper / query
3. 确认组件 props
4. 再落页面
5. 最后验收路由与视觉
```

不得直接在页面中硬编码分类逻辑。

# Phase 19：语义化像素封面系统

## 19.1 阶段目标

把当前重复的像素装饰图升级为有规则、有含义、可复用的封面系统。

当前问题：

```txt
Home / Projects / Lab 等页面的像素图形重复
图形没有实际含义
美观上容易单调
```

目标：

```txt
像素图形由内容元数据生成或映射
不同内容类型有不同模板
同一模板内部根据 metadata 产生差异
图形既统一又不重复
```

## 19.2 必读文档

```txt
docs/00_项目开发计划书.md
docs/standards/03_视觉风格规范.md
docs/standards/04_内容与文件管理规范.md
docs/standards/05_模块化架构规范.md
docs/standards/06_组件设计规范.md
docs/standards/15_数据模型与内部接口规范.md
docs/phases/19_Phase10_全站配色与视觉层级方案.md
```

## 19.3 封面模板建议

定义 5 类模板：

```txt
File Matrix：文件块矩阵，适合 Projects
Terminal Strip：终端条形图，适合 Notes / logs
Save Chip：存档芯片，适合 Projects / Home save slots
Timeline Fragment：时间片段，适合 Archive
Experiment Grid：实验网格，适合 Lab
```

## 19.4 语义映射规则

### Projects

```txt
项目类型 → 模板
项目状态 → 角标 / 状态块
技术栈数量 → 方块数量
进度 / 阶段 → 条形长度
featured → 更强的封面层级
```

### Notes

```txt
category → 主色
文章长度 → 横线数量
type：article/log/note → icon / badge
updated → 轻量状态点
```

### Lab

```txt
experimentType → 模板结构
status → 状态块
coverSeed → 图形变体
视觉实验 → 藕荷 / 紫系块
UI 实验 → 绿系块
交互实验 → 朱红 / 琥珀块
```

### Archive

```txt
type → 节点图形
milestone → 强节点
project-log → 文件片段
release → 版本块
content-log → 文档块
```

## 19.5 技术实现原则

优先使用：

```txt
纯 Astro 组件
CSS Grid
CSS variables
少量 SVG
确定性 seed helper
```

避免：

```txt
canvas
第三方图表库
随机运行时生成导致构建不可控
大图片资产
复杂插画
```

## 19.6 组件建议

```txt
PixelCover.astro
PixelMatrix.astro
PixelBars.astro
PixelCorner.astro
PixelStatusMark.astro
```

Props 建议：

```ts
type PixelCoverProps = {
  variant: 'file-matrix' | 'terminal-strip' | 'save-chip' | 'timeline-fragment' | 'experiment-grid';
  section: 'projects' | 'notes' | 'archive' | 'lab' | 'home';
  status?: string;
  tags?: string[];
  seed?: string;
  compact?: boolean;
};
```

## 19.7 输出物

```txt
语义化 PixelCover 组件族
Projects 封面替换
```

# Phase 20：侧边辅助系统与克制动效

## 20.1 阶段目标

系统化利用左右空白，并加入少量符合 Pixel Archive 气质的动效。

本阶段不追求炫技，只做增强体验与氛围的低干扰改进。

## 20.2 必读文档

```txt
docs/00_项目开发计划书.md
docs/standards/03_视觉风格规范.md
docs/standards/06_组件设计规范.md
docs/standards/09_验收与测试规范.md
docs/phases/19_Phase10_全站配色与视觉层级方案.md
```

## 20.3 侧边系统建议

按页面类型决定侧边内容：

```txt
Home：archive ruler / 坐标 / 状态标记
Projects：项目类型 / 技术栈 / 年月索引
Notes 列表：category / year.month 索引
Notes 详情：On this page / 阅读进度
Archive：year.month / type 索引
Lab：experiment type / status 索引
About：极简坐标 / 当前版本 / built with
```

## 20.4 侧边组件建议

```txt
SideRail.astro
PageIndex.astro
OnThisPage.astro
YearMonthIndex.astro
SectionRuler.astro
ReadingProgress.astro
```

设计原则：

```txt
低对比
小字号
不抢主内容
桌面显示
移动端折叠或隐藏
不遮挡正文
```

## 20.5 克制动效建议

### 交互动效

```txt
卡片 hover 轻微浮起
像素角标位移 1–2px
链接箭头移动 2px
focus ring 清晰但不刺眼
```

### 静置动效

```txt
READY 小方块慢速 blink
ARCHIVE ONLINE 状态点慢速 pulse
首页 grid 极慢漂移 1–2px
状态栏字符轻微 scan / flicker
像素角标偶发轻微变色
```

### 主题切换

```txt
背景 / 文本 / surface 平滑过渡
不做大面积闪烁
不做全屏动画
```

## 20.6 动效边界

必须支持：

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none;
    transition-duration: 0.01ms;
  }
}
```

禁止：

```txt
复杂滚动动画
粒子动画
视差大片移动
强烈霓虹闪烁
持续吸引注意力的动画
新增 GSAP / Framer Motion / 动画库
```

## 20.7 输出物

```txt
侧边辅助组件系统
不同页面的侧边内容映射
少量 hover / idle motion
prefers-reduced-motion 支持
README / WORKLOG 更新
```

## 20.8 验收标准

```txt
npm run build 通过
桌面端侧边信息可见但不抢内容
Notes 详情页 On this page 可用
Projects / Notes / Archive 索引与内容对应正确
移动端无横向滚动
prefers-reduced-motion 下动画关闭或显著减弱
暗色模式动效不刺眼

```
